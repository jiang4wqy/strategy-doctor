# 项目说明 — AI 策略体检与优化引擎（Strategy Doctor）

> 状态说明：本文保留最初产品构想。当前接口、命令和验收结果以根目录 `README.md`、`docs/DEMO.md` 和 `docs/SUBMISSION.md` 为准。

> 赛道：Trading Infra（赛道二）　|　底座：Bitget Agent Hub　|　形态：开源工具 + Demo

**一句话定位：** 别人造策略，我们造给策略做"体检 + 开方"的引擎——用 Bitget 五大分析师 Skill，进化出最能搞垮一个策略的市场剧情，找出它会在什么情况下、为什么会死，按投资者风格打出风险分，并开出修改处方，同时诚实地告诉你"变稳的代价是少赚多少"。

---

## 一、项目大致思路

### 1. 解决什么痛点

整个黑客松（以及 Bitget Playbook 的所有用户）都在做同一件事：造交易策略 / 交易 Agent。但所有人都面对同一个隐形杀手——**过拟合**：一个在历史上 Sharpe 3.5、回测漂亮的策略，很可能在没见过的行情里直接归零。

现状是：大家有"造策略的工具"，却几乎没有"判断这个策略会不会死、以及怎么救"的工具。我们做的就是这把"铲子"——服务全场所有造策略的人。

### 2. 核心流程：一条 诊断 → 处方 → 复测 的闭环

| 阶段 | 做什么 |
|---|---|
| **① 诊断（红队）** | 红队 Agent 用进化算法主动搜索出历史上没出现过、但最能搞垮这个策略的剧情组合（适应度 = 对策略造成的最大伤害），定位最致命的失效点。 |
| **② 评分（因人而异）** | "失败"不是绝对的——30% 回撤对稳健资金是灾难，对激进杠杆是日常。同一策略换一套权重 + 死亡阈值，对不同风格给出不同的 5 维风险分。 |
| **③ 处方（定向进化）** | 红队诊断出"死在资金费率突刺"，处方层就只去变异相关基因（加资金费率过滤 / 加波动率熔断），而不是瞎改。 |
| **④ 复测（防自我过拟合）** | 生成补丁的剧情与验证补丁的剧情必须分开；输出不是"我帮你改好了"，而是"你多扛住了 X 风险，代价是少赚 Y%"的取舍清单。 |

这条闭环也正好把团队最初构想的 **"基因进化 + 反思回写"** 落地成了有真实用途的机制：原来的"盲目进化"变成了由红队诊断驱动的"定向进化"。

### 3. 技术底层（分工很干净）

| 部件 | 职责 | 谁来做 |
|---|---|---|
| 进化引擎（GA） | 搜索最毒剧情 / 生成定向补丁 | 机械搜索 |
| 红队 LLM | 把多源信息合成"为什么会死"的叙事、决定"该修哪" | Agent 推理 |
| 回测 | 量化每个剧情的伤害、验证补丁是否真有效 | 复用 Bitget 底座 |

**一句话：** LLM 负责"修哪、为什么"，GA + 回测负责"修完到底有没有用"。

## 二、与赛题的关联性

### 1. 为什么是赛道二（Trading Infra）

赛道二要的是"给别人的 Agent 做基础设施 / 数据产品"，官方示例之一就是 "Agent monitoring dashboard + risk scoring"。我们做的正是把 risk scoring 做深——零牵强，定义直接吻合。同时赛道二只要求开源 + README + 易接入，不要求实盘全链路，scope 可控，2 周可交付。

### 2. 为什么贴合 "AI × Crypto Trading" 大主题

我们不是又一个交易策略，而是整个策略生态的**质检与加固层**：Bitget Playbook 负责"生策略"，我们负责"体检 + 开方 + 加固"，二者互补而非竞争，服务全场所有参赛者和 Playbook 用户。

### 3. 为什么"只有 Agent 才能做"

普通程序能算出"最大回撤 = 40%"，但把宏观、链上、新闻、情绪、技术五路信息合成一段"它会在什么剧情、为什么死"的叙事，并据此推理出该改哪个基因——只有 LLM Agent 能做。这是不可替代的 AI 价值，正面回应赛题立意。

### 4. 战略站位：卖铲子

淘金热里，卖铲子的最稳。全场都在造策略，而每个人都怕自己造的是过拟合废物——我们就是那把所有人都需要的铲子。

## 三、相关 Skill 的使用与介绍

项目深度复用 Bitget Agent Hub，而非浅尝。这是平台加分的关键。

### 1. 五大分析师 Skill = 五个攻击维度（结构性承重）

这 5 个官方 Skill 不是装饰，而是红队的"五种杀招来源"，也是风险评分卡的五根轴：

| 官方 Skill | 介绍（能做什么） | 用法 = 攻击维度 | 能制造的杀招 |
|---|---|---|---|
| `sentiment-analyst` | 情绪与持仓：贪婪恐惧、多空比、资金费率 | 情绪与拥挤度攻击 | 资金费率突刺、极端贪婪后反转、多头拥挤踩踏 |
| `macro-analyst` | 宏观与跨资产：美联储政策、BTC vs DXY / 纳指 / 黄金 | 宏观与跨资产攻击 | 美联储鹰派意外、DXY 飙升、相关性翻转 |
| `market-intel` | 链上与机构：ETF 流向、巨鲸动向、DeFi TVL | 链上与机构流动性攻击 | ETF 大额流出 + 巨鲸砸盘 + TVL 抽干 |
| `news-briefing` | 新闻聚合与叙事合成：简报、关键词检索 | 叙事与黑天鹅攻击 | 检索历史黑天鹅（暴雷 / 黑客 / 监管）做事件冲击复盘 |
| `technical-analysis` | 技术分析：6 大类 23 个指标 | 技术面体质攻击 | 识别策略适配形态，构造相反形态（趋势↔震荡、波动率切换） |

多数队伍只会浅用 1–2 个 Skill；我们把 5 个全部做成承重的攻击维度，并对应到评分卡的 5 根轴——这正是赞助方最想看到的平台深度使用。

### 2. 其余两层基建的用法

- **Tools（58 个交易 API）：** 拉取历史 K 线、行情、持仓等数据，喂给回测与攻击引擎，无需自写 API 封装。
- **MCP Server（一行接入 Claude / Cursor）：** 开发期直接在 AI 编程工具里调用 Bitget 能力，加速开发。
- **Bitget Playbook（成品平台）：** 作为策略来源 + 回测底座复用——不重造回测引擎，把 Playbook 生成的策略当作"被体检的对象"。

### 3. 安全前提

全程使用只读（Read-only）API Key + 子账号 + 回测 / 模拟盘，不接入实盘真钱自动下单——一个还在做对抗测试的实验性系统，绝不触碰真实资金。
